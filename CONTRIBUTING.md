# Contributing Guide

Thanks for contributing!

## Development steps

1. Fork and clone the repository.
2. Create a branch from `develop`:
   - `feature/<name>`
   - `fix/<name>`
3. Keep commits focused and descriptive.
4. Open a pull request with clear context and screenshots for UI changes.

## Commit convention (recommended)

- `feat:` new feature
- `fix:` bug fix
- `docs:` documentation only
- `style:` formatting-only changes
- `refactor:` code refactor without behavior change
- `chore:` maintenance chores

## Pull request checklist

- [ ] The change is focused and minimal.
- [ ] Documentation is updated if needed.
- [ ] No obvious console errors in browser.
- [ ] UI changes include before/after notes.
